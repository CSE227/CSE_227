DOM-Based XSS Demonstration

1. Try the Exploit

This search bar is deliberately vulnerable to DOM-based Cross-Site Scripting (XSS). To test it, copy the malicious payload below and paste it into the search input.

Malicious Payload:
<img src="x" onerror="alert('DOM XSS Executed!')">

2. How the Vulnerability Works

DOM XSS occurs when an application contains client-side JavaScript that processes data from an untrusted source in an unsafe way, usually by writing that data to a dangerous sink.

  • The Source: In this page, the JavaScript reads the ?q= parameter directly from the website's URL using window.location.search.
  • The Sink: The script then takes that exact input and inserts it into the page using innerHTML. Because innerHTML parses text as executable HTML, the browser attempts to render the image tag.
  • The Execution: The browser tries to load an image from the source "x". When this fails, it triggers the onerror event, executing our JavaScript alert.

3. How to Fix It

To patch this, you must ensure the browser treats the user's input strictly as text, not as code. You do this by replacing dangerous sinks like innerHTML with safe alternatives like textContent.

// INSECURE (What this page is doing): resultsDiv.innerHTML = "You searched for: " + searchQuery; // SECURE (How to fix it): resultsDiv.textContent = "You searched for: " + searchQuery;
